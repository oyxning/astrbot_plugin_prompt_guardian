# Prompt Guardian · AstrBot 提示词守卫

![License](https://img.shields.io/badge/License-AGPL--3.0-red.svg)
![PTD Core](https://img.shields.io/badge/PTD-1.0-brightgreen.svg)
![Version](https://img.shields.io/badge/Version-v1.0.0-blue.svg)

> **Prompt Guardian (原 Anti-Prompt Injector)** 是 AstrBot 的下一代提示词安全插件。经过全面重构，v1.0.0 带来了更轻量的异步架构、更强的本地启发式检测能力以及更低的大模型依赖。

---

## ✨ v1.0.0 重构亮点

- **⚡ 极速响应**：全新设计的异步 WebUI (基于 `asyncio`)，资源占用极低，不再阻塞主线程。
- **🛡️ 本地优先防御**：升级版 PTD Core v1.0 引擎，针对高风险攻击（如 JSON 伪造、越狱模式）实现**毫秒级本地阻断**，不再完全依赖大模型。
- **🔒 安全加固**：新增 HMAC 签名锁定机制，防止提示词在处理链路中被篡改；支持输入清洗 (`sanitize`) 功能。
- **🧩 模块化架构**：代码结构清晰分离（核心逻辑/检测引擎/WebUI服务），便于维护与二次开发。

---

## 🛡️ 防御模式说明

Prompt Guardian 提供四种通俗易懂的防御模式，满足不同场景的安全需求：

| 模式名称 | 英文标识 | 安全等级 | 适用场景 | 说明 |
| :--- | :--- | :--- | :--- | :--- |
| **观察模式** | `passive` | 🟢 低 | 测试/调试 | **仅记录日志，不拦截任何请求**。适合刚安装插件时，观察正常业务是否会触发误报。 |
| **标准模式** | `standard` | 🔵 中 | 日常使用 | **拦截高风险请求**，中风险请求放行。平衡了安全性与用户体验，适合大多数群聊场景。 |
| **严格模式** | `strict` | 🟠 高 | **推荐默认** | **拦截高风险和中风险请求**。确保绝大多数潜在威胁无法触达 LLM，误报率极低。 |
| **激进模式** | `aggressive` | 🔴 极高 | 高危环境 | **拦截所有可疑请求并自动拉黑攻击者**。宁可错杀不可放过，适合公开接口或遭受攻击时开启。 |

> **提示**: 默认启用 **严格模式 (Strict)**，兼顾安全与稳定。您可以通过 `/切换防护模式` 指令或 WebUI 随时调整。

## 🕹️ WebUI 控制台

访问 `http://127.0.0.1:18888` (默认端口) 即可进入仪表盘。

- **实时监控**: 查看拦截统计、最近的威胁日志。
- **配置管理**: 在线调整防御模式、阈值与黑白名单。
- **审计日志**: 详细记录每次拦截的触发原因、评分与原始提示词。
- **安全认证**: 支持密码登录与会话管理。

## ⚙️ 配置说明

插件首次运行会自动生成默认配置。主要配置项如下：

```json
{
  "enabled": true,
  "defense_mode": "strict",          // 默认: strict (严格模式)
  "sanitize_enabled": true,          // 默认: true (启用输入清洗)
  "enable_signature_lock": true,     // 默认: true (启用防篡改签名)
  "webui_enabled": true,             // 是否开启 WebUI
  "webui_port": 18888,               // WebUI 端口
  "review_provider": "",             // 审计用 LLM 供应商 (留空使用默认)
  "review_model": "",                // 审计用 LLM 模型
  "auto_blacklist": true,            // 是否自动拉黑攻击者
  "blacklist_duration": 60           // 拉黑时长 (分钟)
}
```

## 🚀 升级/安装指南

1.  **备份**: 如果您正在使用旧版 (AntiPromptInjector)，请备份 `config.json` (插件会自动迁移兼容配置)。
2.  **安装**: 将插件文件夹重命名为 `astrbot_plugin_prompt_guardian` (推荐) 并放入 `plugins/` 目录。
3.  **重启**: 重启 AstrBot，观察日志确认加载成功。
